# 로그라이크 RPG 마스터 계획 파일

## 문서 구성

- 상세 시스템: `roguelike-rpg-plan-part1-system.md`
- 콘텐츠 표: `roguelike-rpg-plan-part2-content.md`
- 제작 사양서: `roguelike-rpg-plan-part3-implementation-spec.md`

이 파일은 위 3개 문서를 합쳐서 빠르게 검토할 수 있도록 정리한 통합본이다.

## 1. 핵심 결정 요약

- 주인공은 `루미` 고정
- 시작 스킬은 `기본 공격`만 보유
- 게임 시작 직후 `스킬 드래프트 3회`
- 첫 드래프트는 `매직 가드 / 배리어 / 가드 / 회피 태세` 풀에서 3개 제시
- 이후 드래프트는 `궁극기 제외 전체 스킬`에서 3개 제시
- 스킬, 아티팩트, 상점은 모두 `리셋 2회 + 영어 퀴즈` 규칙 공유
- 난이도는 `Easy / Normal / Hard`
- 랜덤 인카운트 제거, `선형 진행` 채택
- 보스 승리 후마다 상점 출현
- `Lv 5 / 10 / 15 / 20`에 아티팩트 선택
- `천계 바로 전`에 이벤트 던전 1개 랜덤 진행
- 이벤트 던전 승리 시 `기사 / 사제 / 암살자 / 마법검사 / 대현자` 전직 가능
- 전직 시 `해당 직업 궁극기 2개` 자동 습득
- 상태 UI는 `적용 중인 상태만 표시`

### 방어 과성장 해결

- 레벨업 성장에서 `DEF/MDEF 증가율 0%`
- 방어 체감은 `장비`, `방어형 스킬`, `아티팩트`로만 만든다.
- 피해 공식은 감쇠 상한 `65%`를 둬서 후반 방어 무력화를 막는다.

## 8. 이번 안에서 고정한 애매한 항목

### 8-1. 무기 제한

- 하드 제한 대신 `소프트 친화도` 채택
- 추천 무기: `+12%`
- 비추천 무기: `-15%`
- 비추천 궁극기: `-25%`

### 8-2. 퀴즈 실패 시 리셋 처리

- 정답: 전체 재롤
- 오답: 1개 부분 재롤
- 공통: 리셋 횟수 1회 소모

### 8-3. 과도한 스킬 수 문제

- `보유 스킬`과 `장착 스킬` 분리
- 일반/보조 장착 8칸, 궁극기 2칸

### 8-4. 영어 퀴즈의 사용 위치

- 유지
- 단, 전투 승리 보너스 퀴즈는 제거 권장
- 퀴즈는 `스킬/아티팩트/상점 리셋`에 집중

## 10. 구현 착수 순서

1. `roguelike_data.js`와 `roguelike.html` 뼈대를 먼저 만든다.
2. 전투 엔진과 보스 AI 래퍼를 붙여 1장~2장까지 작동시킨다.
3. 시작 드래프트, 레벨업 드래프트, 상점 재롤을 붙인다.
4. 이벤트 던전과 전직 재계산을 붙인다.
5. 히든 해금과 세이브/로드를 붙인다.
6. `npm run verify`와 로그라이크 스모크 테스트를 추가한다.
