# 로그라이크 RPG 기획서 Part 1. 시스템/밸런스 코어

## 1. 목표

- 기존 `english_vocab_version`의 스킬, 아티팩트, 영어 퀴즈, 보스 AI를 최대한 재활용한다.
- 플레이어 캐릭터는 `루미` 1명으로 고정한다.
- 상점 구매형 스킬 구조를 제거하고, `3지선다 드래프트` 중심의 로그라이크 흐름으로 바꾼다.
- 랜덤 인카운트를 없애고 `선형 진행 + 보스 후 상점 + 레벨업 드래프트` 구조로 정리한다.
- 기존 후반 방어력 과잉 문제를 해결하기 위해 `방어/마방은 레벨업으로 증가하지 않게` 설계한다.

### 2-2. 레벨업 성장 공식

- 최대 레벨: `20`
- 레벨업마다 `기본 공격 + 1개 스킬 드래프트`
- 레벨업 성장치는 `초기 베이스 스탯` 기준으로 고정 계산한다.
- 권장 성장식:

```text
HP 증가량   = floor(baseHp   * 0.055 * jobWeight.hp)
MP 증가량   = floor(baseMp   * 0.040 * jobWeight.mp)
ATK 증가량  = floor(baseAtk  * 0.045 * jobWeight.atk)
MATK 증가량 = floor(baseMatk * 0.045 * jobWeight.matk)
DEF 증가량  = floor(baseDef  * 0.000 * jobWeight.def)
MDEF 증가량 = floor(baseMdef * 0.000 * jobWeight.mdef)
```

- 핵심 결정:
  - `DEF/MDEF 성장률은 0%`
  - 방어 체감은 `장비`, `가드류 스킬`, `배리어류`, `아티팩트`로만 확보
  - 이렇게 해야 기존 `레벨 15 이후 방어 과포화`를 원천 차단할 수 있다.

### 2-3. 전직 후 레벨업 분 일괄 재적용

- 이벤트 던전 클리어 후 전직 시점에 `레벨 2부터 현재 레벨까지의 누적 성장분`을 새 직업 가중치로 다시 계산한다.
- 즉, 전직은 단순히 현재 스탯에 보너스를 더하는 방식이 아니라 `성장 로그 전체를 재빌드`하는 방식으로 처리한다.
- 구현 함수 권장안:

```text
rebuildLevelGrowth(playerBaseStats, level, jobId)
```

## 7. 스킬 드래프트 규칙

### 7-1. 시작 드래프트

- 게임 시작 시 `3회 연속 드래프트`
- 시작 시 보유 스킬은 `기본 공격`만 있음

1. 첫 드래프트:
   - 풀: `매직 가드`, `배리어`, `가드`, `회피 태세`
   - 이 중 `3개 랜덤 제시`, 1개 선택
2. 두 번째 드래프트:
   - 풀: 궁극기 제외 전체 스킬
   - 이미 보유한 스킬 제외
3. 세 번째 드래프트:
   - 풀: 궁극기 제외 전체 스킬
   - 이미 보유한 스킬 제외

### 7-2. 레벨업 드래프트

- 매 레벨업마다 `스킬 1회 드래프트`
- 옵션은 항상 `3개`
- 중복 제시는 금지
- 궁극기는 전직 보상에서만 습득

### 7-3. 직업 노출 규칙

- 권장안: 현재 보유 스킬에서 `대표되지 않은 직군` 우선 노출
- 계산 방식:
  - `ownedSkills`에서 jobTag 집계
  - 미보유 직군 스킬 가중치 `65`
  - 이미 보유한 직군 스킬 가중치 `35`
- 전직 후에는 현재 직업 스킬도 계속 노출되되, 직업 편중을 막기 위해 `동일 직군 2개 초과 동시 제시 금지`

### 7-4. 리셋(재롤) 규칙

- 스킬/아티팩트/상점 모두 공통 규칙 사용
- `선택 1회당 리셋 2회 가능`
- 리셋 시 영어 퀴즈 출제

| 결과 | 처리 |
| --- | --- |
| 정답 | 3개 옵션 전체 재롤 |
| 오답 | 3개 중 1개만 교체 |
| 공통 | 리셋 횟수 1회 소모 |

- 이유:
  - 영어 학습 보상을 유지
  - 오답 시 진행이 완전히 막히지 않음
  - 드래프트 흐름이 과도하게 끊기지 않음

### 7-5. 전투용 장착 스킬 수 제한

- 누적 습득 스킬은 계속 쌓이지만, 전투 버튼 과밀 방지를 위해 `장착 스킬` 개념을 둔다.
- 권장 슬롯:
  - 기본 공격 1칸 고정
  - 일반/보조 스킬 `8칸`
  - 궁극기 `2칸`
- 새 스킬 획득 시:
  - 장착 여유가 있으면 자동 장착
  - 여유가 없으면 `장착/비장착 교체 팝업` 표시

## 8. 물리/마법 밸런스 해결안

### 8-1. 문제 정의

- 기존 자스민처럼 `한쪽 공격 스탯만 높은 캐릭터`는 반대 타입 스킬의 체감이 깨졌다.
- 이번 버전은 루미 고정이지만, 랜덤 드래프트로 여러 직업 스킬을 섞기 때문에 동일 문제가 재발할 수 있다.

### 8-2. 권장 해결안: 무기 친화도 + 혼합 스케일

- 하드 락(착용 불가)보다 `소프트 패널티`가 로그라이크에 안전하다.
- 공격식 권장안:

| 스킬 태그 | 계산식 |
| --- | --- |
| Physical | `(ATK x 1.00) + (MATK x 0.20)` |
| Magical | `(MATK x 1.00) + (ATK x 0.20)` |
| Hybrid | `(ATK x 0.65) + (MATK x 0.65)` |
| Support | 직접 피해 없음 |

- 무기 친화도 권장안:

| 상태 | 배율 |
| --- | ---: |
| 추천 무기 일치 | +12% |
| 중립 무기 | 0% |
| 비추천 무기 | -15% |
| 궁극기 비추천 무기 | -25% |

- 이렇게 하면:
  - 물리 스킬이 완전히 죽지 않음
  - 무기 선택 의미가 살아남
  - 랜덤 스킬 획득이 빌드 브릭으로 이어지지 않음

## 9. 피해 공식

### 9-1. 신규 권장 피해 공식

```text
attackPower = skillPower * scaledOffense
mitigation = min(0.65, defense / (defense + 120 + targetLevel * 8))
damage = round(attackPower * (1 - mitigation) * random(0.94 ~ 1.06))
```

### 9-2. 의도

- 방어가 높아도 감쇠율 상한을 `65%`로 막아 후반 무적화를 방지
- 레벨과 적 구간에 따라 분모가 조금씩 커져 특정 장비 1개로 난이도가 붕괴하지 않음
- 현재 `100 / (100 + def)` 공식을 그대로 쓰는 것보다 방어 곡선이 완만해짐
