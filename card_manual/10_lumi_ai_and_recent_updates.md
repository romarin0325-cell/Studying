# 10편 Lumi AI·학습 시스템·최근 변경점

## 1. 시스템 분리

현재 AI 관련 기능은 4갈래다.

1. 루미 개인과외
2. 루미 질문하기(일반 채팅)
3. TOEIC 해설 후 질문하기
4. 루미 데이트

이들 모두 `card/api.js`를 공유하지만, 프롬프트와 모델/검색 설정은 다르다.

## 2. 모델 체계

현재 선택 가능한 모델은 3개다.

| 라벨 | 모델 ID | 특성 |
|---|---|---|
| Pro | `gemini-2.5-pro` | 검색 가능, thinking budget 사용 |
| Flash | `gemini-3-flash-preview` | 검색 가능, flashLike |
| Flash Lite | `gemini-3.1-flash-lite-preview` | 검색 불가, flashLike |

### 공통 규칙

- 일반 질문은 검색 ON/OFF 토글을 가진다
- TOEIC 질문은 세션 정보 우선 답변, 필요 시에만 검색 보조
- Flash 계열은 재시도 2회, Pro는 1회
- 요청 타임아웃은 120초

## 3. 개인과외 프롬프트

`getTutoringContent()`는 아래 구조로 프롬프트를 만든다.

1. `LUMI_PERSONA`
2. 오해 유발 이벤트 지시문(30%, 30스테이지 이상 50%)
3. `LECTURE_FORMAT`
4. 대상 단어 또는 숙어 정보

### 현재 개인과외 강의 형식

- 도입
- 본문
- 암기 비법
- 용법과 뉘앙스
- TOEIC 스타일 예문
- 심쿵 포인트
- 마무리 멘트

### PR #320 + PR #329 반영 핵심

암기 비법 작성 규칙은 현재 아래 우선순위를 따른다.

1. 억지 발음 말장난 금지
2. 자연스러운 연상 우선
3. 자연 연상이 약하면 어원(prefix/root) 기반 논리 설명
4. 그것도 약하면 루미와 형아의 장면 묘사로 연결
5. `trap_word`가 있으면 해당 페어 단어와의 차이를 위주로 설명

즉, PR #329 이후에는 단순히 “단어를 재미있게 외우게 한다”가 아니라 “헷갈리는 페어 단어와 어떻게 다른지를 중심으로 외우게 한다”가 최신 규칙이다.

## 4. 질문하기 UI

일반 질문 모달은 아래 버튼을 가진다.

- 모델 전환
- 검색 ON/OFF
- 응답 취소
- 대화 초기화
- 나가기

응답에는 아래 메타데이터가 붙을 수 있다.

- 사용 모델
- 검색어 목록
- grounding source 링크
- 재시도 버튼
- 대체 모델로 재시도 버튼

## 5. TOEIC 질문 세션

TOEIC 질문은 일반 채팅과 다르게 “방금 푼 세트의 지문/문항/보기/정답/해설”을 컨텍스트로 먼저 넣는다.

규칙:

- 현재 세션 정보로 먼저 답변
- 부족할 때만 검색 보조
- Flash 계열일 때는 최근 히스토리 6개만 유지
- Flash 계열 TOEIC 질문은 생각 수준을 `medium`으로 낮춘다

## 6. 데이트 시스템

- 별도 `DATE_LUMI_PERSONA`와 `DATE_FORMAT` 사용
- 30% 확률로 오해 유발 지시문 추가
- 현재 데이트는 `gemini-3-flash-preview` 고정 호출

## 8. 문서화 시 결론

현재 버전의 Lumi 문서는 아래처럼 이해하면 된다.

- 일반 질문: 검색 가능 지식형 루미
- 개인과외: 단어 암기/오해 이벤트 중심의 강의형 루미
- TOEIC 질문: 세션 컨텍스트 우선의 해설형 루미
- 데이트: 별도 연출형 루미

구문서에는 PR #320~#329 구간의 프롬프트 보정과 UI 제어 변화가 없으므로, 최신 매뉴얼에서는 이 구간을 별도 항목으로 다루는 것이 필수다.
