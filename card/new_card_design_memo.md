# 신규 카드 기획 메모

본 문서는 추후 추가될 예정인 신규 카드 3종(베놈, 대일여래, 기억도살자)에 대한 기획 및 평가 메모입니다.

## 1. 베놈 (Venom)
- **속성**: 땅 (Nature)
- **등급**: 전설 (Legend)
- **역할**: 디버퍼 (Debuffer)
- **스탯**: 에인션트 드래곤과 비슷 (잠정: HP 530, ATK 115, MATK 95, DEF 70, MDEF 70)
- **특성**: 사망시 400% 물리대미지 + 부식
- **스킬**:
  - `가드` (1티어 / 보조): 대미지 반감
  - `플래이그` (2티어 / 마법): 5턴에 걸쳐 1배 마법 대미지, 매 타격시 랜덤 디버프 (약화, 침묵, 부식, 저주, 암흑, 디바인, 작열, 유혹 중 하나) 부여 (팬텀의 나이트메어와 유사한 핸들러)
  - `베놈허그` (3티어 / 물리): 물리 2.5배율, 적에게 암흑 부여

### 📊 기획안 평가
- **상위호환 여부**: 동일 등급/속성의 디버퍼인 '에인션트 드래곤'은 사망 시 5종의 디버프를 거는 극단적 디버퍼입니다. 베놈은 디버프와 함께 데미지 기여(사망시 물리 400%, 플래이그 지속 대미지)에 조금 더 치중한 형태이므로 완벽한 상위호환이라기보다는 공격적인 디버퍼 포지션을 가집니다.
- **밸런스**: '플래이그'가 매 턴 타격마다 디버프를 걸게 된다면, 총 5번의 디버프 중첩과 대미지가 들어가므로 2티어 스킬 치고 스노우볼링 포텐셜이 매우 높습니다. 약간 오버밸런스가 될 소지가 있어, 총 대미지량이나 타격 횟수 조정이 필요할 수 있습니다.
- **컨셉의 재미**: 맹독/역병이라는 컨셉에 걸맞게 지속적으로 적을 좀먹으면서 상태이상을 유발하는 스킬 구성이 테마와 아주 잘 어울리며 전투의 재미를 더해줍니다.
- **구현 난이도 (보통)**:
  - 사망 시 특성은 기존 사망 핸들러(`death_dmg_phy`와 디버프 부여)를 결합하여 새 특성을 만들면 되므로 비교적 쉽습니다.
  - `플래이그`는 팬텀의 지연 타격(`phantom_nightmare` 등)을 응용하되, 매 타격마다 디버프 풀에서 랜덤으로 효과를 골라 적용하는 전용 지연 공격 핸들러(`delayed_attack_with_random_debuff` 등)의 추가 개발이 필요합니다.

## 2. 대일여래 (Dainichi Nyorai)
- **속성**: 빛 (Light)
- **등급**: 전설 (Legend)
- **역할**: 밸런서 (Balancer)
- **스탯**: 베히모스와 비슷 (잠정: HP 560, ATK 115, MATK 90, DEF 75, MDEF 55)
- **특성**: 사망시 모든 필드버프를 삭제하고 대지의축복 부여 (※ 구현 시 handleDeathTraits에 필드버프 일괄 삭제 기능 추가 필요)
- **스킬**:
  - `오지관정` (1티어 / 보조): 대미지 반감 + 랜덤 필드버프 부여
  - `파쇄륜` (3티어 / 물리): 물리 2배율, 모든 필드버프 제거 후 제거한 수 × 3배율 위력 증가 (피날레 핸들러 방식)
  - `일체지광` (2티어 / 마법): 마법 2배율, 적에게 디바인 부여

### 📊 기획안 평가
- **상위호환 여부**: 빛 속성 전설 밸런서 포지션 자체가 유니크하며, 기존 카드들과 비교해 '필드버프 자체를 자원으로 쓰거나 초기화'하는 완전히 독보적인 메커니즘을 가집니다.
- **밸런스**: 루미 시리즈 등 필드버프를 겹겹이 쌓는 조합과 시너지를 낼 때 강력합니다. 다만 `파쇄륜`은 필드버프가 2개만 지워져도 기본 2배율 + 6배율 = 총 8배율의 물리 데미지가 나오며, 최대치(필드버프 5개 기준 17배율)일 경우 보스전 밸런스를 크게 붕괴시킬 수 있습니다. 제거 개수당 배율 증가폭을 1.0~1.5 정도로 하향 조정할 필요가 있습니다.
- **컨셉의 재미**: 공들여 쌓은 필드버프를 한순간에 터뜨려 엄청난 폭딜을 내거나, 적이 깔아둔 불리한 필드버프를 싹 청소해버리는 '리셋/초기화'의 쾌감이 훌륭합니다.
- **구현 난이도 (보통)**:
  - 필드 버프 일괄 삭제는 기존 `logic.js`에 `clear_all_field_buffs` 관련 핸들러를 새로 추가하면 쉽게 구현 가능합니다.
  - `파쇄륜`의 경우 전투 런타임에서 현재 존재하는 필드 버프 개수를 카운트한 뒤 삭제하고, 그 수에 비례하여 대미지 배율을 뻥튀기하는 특수 스킬 이펙트 처리가 요구됩니다.

## 3. 기억도살자 (Memory Slaughterer)
- **속성**: 어둠 (Dark)
- **등급**: 전설 (Legend)
- **역할**: 딜러 (Dealer)
- **스탯**: 그레이와 동일하나 물리 공격력 15 추가 (잠정: HP 480, ATK 160, MATK 125, DEF 65, MDEF 65)
- **특성**: 아군 전체의 특성을 무효화한다.
- **스킬**:
  - `회피태세` (1티어 / 보조): 회피율 50% 증가
  - `스티그마` (3티어 / 물리): 물리 2.5배율, 13턴 이후 사용 시 대미지 2배
  - `메모리길로틴` (3티어 / 마법): 마법 5배율, 사용 시 자신 1턴 스턴

### 📊 기획안 평가
- **상위호환 여부**: 같은 어둠 전설 딜러인 '그레이'는 관통과 랜덤 폭딜(차원절단/영혼절단)로 승부하는 반면, 기억도살자는 파티 전체에 큰 페널티를 주는 대신 높은 깡스탯과 확정적인 고배율 스킬로 찍어누르는 타입입니다. 상위/하위 호환보다는 완전히 다른 운영을 요구합니다.
- **밸런스**: '아군 전체 특성 무효화'라는 페널티는 덱 빌딩 게임에서 뼈아픈 디메리트입니다. 그에 비해 스킬의 리턴이 살짝 애매할 수 있습니다. 13턴 이후 발동하는 `스티그마`의 조건이 꽤 까다로우며, 5배율 스킬에 자신 스턴이 달린 것도 리스크가 큽니다. 높은 물공(160)에 비해 주력 폭딜기(`메모리길로틴`)가 마법 판정이라는 점도 딜레마를 줍니다. 하이리스크 포지션인 만큼 대미지 배율을 소폭 상향하거나 조건부 턴을 완화하는 것을 고려해 볼 만합니다.
- **컨셉의 재미**: 강력한 존재감으로 인해 파티원의 특성마저 집어삼킨다는 역발상적이고 파괴적인 컨셉이 매우 인상적입니다. 특정 특성이 오히려 독이 되는 기믹성 스테이지가 있다면 역으로 활약할 수 있는 재미난 조커 카드입니다.
- **구현 난이도 (어려움)**:
  - 아군의 모든 특성을 무효화하는 효과는 `logic.js` 등 전투 전반에 걸쳐 큰 수정을 요구합니다. 초기 스탯 계산, 패시브 시너지 연산, 사망 시 특성 발동(`handleDeathTraits`), 피격 시 특성 발동(`handleOnHitTraits`) 등 특성이 관여하는 모든 런타임 구간에서 "파티 내에 '기억도살자'가 살아있을 경우 특성 무시"라는 예외 처리 플래그를 추가해야 하므로 구조적으로 사이드 이펙트 버그가 발생할 위험이 높습니다.

| 리스크 | 설명 | 심각도 |
|---|---|---|
| 스컬드래곤 디메리트 무효화 | `leader_self_atk_party_def_down`은 파티 방/마방을 깎는 디메리트 특성인데, 이것까지 무효화하면 이득이 됨. 특성 억제가 디메리트 특성도 지우므로 의도치 않은 상위호환 발생 | 🔴 높음 |
| 기억도살자 자신의 특성 | "아군 전체 특성 무효화"가 자기 자신에게도 적용되나? 그렇다면 특성 자체가 자기모순. 아니라면 자기 자신은 왜 예외인지 설명 필요 | 🔴 높음 |
| 사망 특성 타이밍 | 기억도살자가 살아있을 때만 억제인지, 사망 후에도 억제가 유지되는지에 따라 완전히 다른 게임 경험. 기억도살자 사망 → 억제 해제 → 나머지 카드 사망 특성 발동 가능? | 🔴 높음 |
| `activeTraits` 배열 오염 | `buildBattlePlayer`에서 이미 `activeTraits`에 push된 시너지 키를 나중에 제거해야 함. 초기화 순서에 민감한 구조 | 🟡 중간 |
| `calculateInitialStats` 이중 호출 | 기억도살자가 덱의 몇 번째냐에 따라 앞 카드는 이미 특성 적용 완료 → 나중에 되돌려야 함 | 🟡 중간 |
| 향후 신규 특성 추가 시 | 새 특성을 추가할 때마다 기억도살자 예외 처리를 잊으면 버그 발생. 영구적 유지보수 부담 | 🔴 높음 |
| `chaos_blessing_double` | 앤트로피의 혼돈 축복 2배 효과는 `buildBattlePlayer` 내부에서 처리됨 (`battle_runtime.js:192`). 이것은 `calculateInitialStats` 외부이므로 별도 억제 필요 | 🟡 중간 |

**부분적으로 예측 가능한 영역**
- `calculateStats`와 `calculateDamage`의 특성 분기: 여기에 `traitsSuppressed` 플래그를 체크하는 guard clause를 넣으면 처리 가능하나, 산재된 `if (trait && trait.type === ...)` 패턴 36개 이상에 모두 추가해야 함.
- `executeSkill`의 일반공격 추가효과: `source.proto.trait.type` 체크가 5곳에 하드코딩되어 있음.

#### 3. 구현 방식 비교

**방식 A: 글로벌 플래그 (`traitsSuppressed`)**
- 전투 초기화 시:
  ```javascript
  if (덱에 memory_slaughterer 존재)
    rpg.battle.traitsSuppressed = true;
  ```
- 모든 특성 체크 구간에서:
  ```javascript
  if (rpg.battle.traitsSuppressed) return;
  ```
- **장점**: 개념적으로 단순
- **단점**: 36+ 코드 경로에 guard clause 삽입 필요. `calculateInitialStats`는 `rpg.battle` 컨텍스트를 받지 않으므로 시그니처 변경 필요. 신규 특성 추가 시 guard clause 누락 리스크.
- **변경 파일**: `logic.js` (~30곳), `battle_runtime.js` (~20곳)
- **예상 diff**: +100~150줄
- **리스크**: 🔴 높음 — 누락 시 사일런트 버그

**방식 B: `proto.trait`을 더미로 교체**
- 전투 초기화 시:
  ```javascript
  if (덱에 memory_slaughterer 존재)
    모든 플레이어의 proto.trait을 { type: 'none', desc: '특성 억제됨' }으로 교체
  ```
- **장점**: 기존 코드 수정 최소화. 모든 특성 분기가 자연스럽게 무시됨.
- **단점**: `proto`는 원본 카드 데이터 객체를 참조하므로, 직접 수정하면 전역 데이터 오염. 깊은 복사가 필요하며 이 경우 `proto` 참조 일관성이 깨짐. 사망 특성, 피격 특성도 전부 무효화되어 "사망 시 반격" 같은 기대 행동이 사라짐 (이것이 의도된 것인지 확인 필요).
- **리스크**: 🟡 중간 — 깊은 복사 비용 + `proto` 참조 오염 위험

**방식 C: 특성을 "약화" 수준으로 기획 변경 ⭐ 권장**
**"아군 전체 특성 무효화" 대신 "아군 시너지/패시브 스탯 보정만 무효화"**로 축소합니다.
- 구현:
  ```javascript
  calculateInitialStats에서 deck에 memory_slaughterer가 있으면
    시너지 테이블(syn_*) 스킵
    파티 스탯 부스트(party_stat_boost, syn_*_party_*) 스킵
  ```
  사망 특성, 피격 특성, 전투 중 트리거 특성은 유지
- **장점**: 수정 범위가 `calculateInitialStats` 1개 함수로 한정. 사망/피격 특성은 건드리지 않으므로 런타임 안정성 유지. "특성 무효화"의 핵심 페널티(스탯 버프 제거)는 보존하면서 사이드이펙트 제어 가능.
- **단점**: 원래 기획의 "완전 무효화"보다 페널티가 약해짐.
- **변경 파일**: `logic.js` (1개 함수), `battle_runtime.js` (초기화 부분 일부)
- **예상 diff**: +20~30줄
- **리스크**: 🟢 낮음

#### 4. 스킬 분석 (부수적)
스킬 자체는 기존 엔진으로 문제없이 구현 가능합니다.

| 스킬 | 구현 방식 | 기존 선례 |
|---|---|---|
| 회피태세 | `{ type: 'buff', id: 'evasion', duration: 1 }` | 루나 등 다수 |
| 스티그마 (2.5x, 13턴 이후 2배) | `{ type: 'dmg_boost_turn_limit' }` 역방향 변형, 또는 새 핸들러 `dmg_boost_after_turn` 추가 | `dmg_boost_turn_limit` 존재 |
| 메모리길로틴 (5x + 자신 스턴) | `val: 5.0` + `{ type: 'self_debuff', id: 'stun' }` | 루나의 다크메테오 (4.5x + 스턴)와 동일 패턴 |

> **NOTE**
> 스티그마의 "13턴 이후 배율 2배"는 기존 `dmg_boost_turn_limit`의 역조건 (N턴 이내 → N턴 이후)이므로 `dmg_boost_after_turn` 핸들러를 새로 추가하면 됩니다. 3줄 내외의 변경으로 구현 가능합니다.

- **컨셉의 재미**: 축적된 신성한 힘(디바인)을 한 번에 기폭시키고 장렬하게 산화하는 '홀리밤' 컨셉이 직관적이며 전투의 재미를 더해줍니다.

## 6. 빅토리아 (Victoria)
- **속성**: 빛 (Light)
- **등급**: 전설 (Legend)
- **역할**: 버퍼 (Buffer)
- **스탯**: 피닉스와 비슷 (잠정: HP 510, ATK 130, MATK 90, DEF 75, MDEF 70)
- **특성**: 대장 배치일 경우 필드버프의 효과를 2배로 받는다. (자신한정)
- **스킬**:
  - `생츄어리` (2티어 / 보조): 3턴간 대미지 반감
  - `에태르랜스` (2티어 / 마법): 마법 2배율, 적에게 부식 부여
  - `기적의증명` (3티어 / 보조): 3턴 지연 후 필드버프 여신강림+트윙클파티 부여

### 📊 기획안 평가
- **상위호환 여부**: 방어, 디버프, 그리고 궁극적인 지연형 버프를 모두 갖춘 하이엔드 버퍼 겸 탱커입니다. 스스로 버프 효율을 극대화한다는 점에서 기존 버퍼들과 확실한 차별화가 됩니다.
- **밸런스**: 3턴 지연이라는 페널티가 있지만, `여신강림`과 `트윙클파티`라는 최고급 필드버프 2개를 동시에 거는 것은 게임 내 최고 수준의 리턴입니다. 또한 대장 배치 시 필드버프 효과를 2배로 받기 때문에, 본인의 생존력과 화력이 크게 상승하여 덱의 핵심 코어로 활약할 수 있습니다. 1티어 스킬 대신 2티어 스킬이 2개 배정된 독특한 스킬셋도 눈에 띕니다.
- **컨셉의 재미**: 성역을 펴고 버티다가 3턴 뒤에 기적을 증명하며 압도적인 버프를 받는 성기사 컨셉입니다. 조건 달성 시 터지는 뽕맛이 엄청납니다.
- **구현 난이도 (어려움)**:
  - 대장(배치 인덱스 2)일 때만 본인 한정으로 필드버프 능력치 상승/감소 스탯 연산을 2배로 적용하는 특수 로직이 요구됩니다.
  - 3턴 지연 후 다중 필드버프를 부여하는 시스템(지연 큐에 스킬 이펙트 2개 예약)은 기존 지연 핸들러의 확장 혹은 새로운 이펙트 타입 생성이 필요합니다.
